Dissertation project

A survival horror game featuring procedurally generated audio. Built in Unity and C# (PC).

This project explores the use of algorithms to generate a horror game soundtrack. This is achieved by sequencing layers of audio samples in real time, creating a unique listening experience for every playthrough. The premise of the game is simple and tasks the player with surviving as long as possible and evading enemies. The game ends when all five lives are lost.

One algorithm decides whether to continue playing a layer of musical instrumentation or transition to a new one. This background audio is sensitive to the surroundings of the player, adapting the soundtrack to suit their environment. A second algorithm layers additional audio samples for variety and added tension.

The audio also varies with the intensity of the situation. The algorithms consider factors such as how far the player is from a threat and what they can see in the game.

Tipping Point

A physics-based arcade game developed collaboratively over the course of 24 hours. Built in Godot and GDScript (PC).

This game was developed for the November 2017 Game Jam at the University of Lincoln. It can be played as a singleplayer experience in which the player assumes control of a cube balancing on a see-saw. The aim is to stay on top as long as possible while avoiding being squashed by falling shapes. A second player can join in and defend the cube by using a cannon to deflect incoming shapes.

Two additional game modes are available to challenge players further.

Our game won best use of the theme.

Pacman clone

A clone of the classic arcade game Pacman built using C++ and SDL2 (PC).

This game makes use of the SDL2, SDL_image, SDL_mixer, and SDL_ttf libraries to provide visuals, audio, and text. Object oriented programming was employed extensively to implement a player character, ghosts, animations, and a grid-based level layout.

As well as the core gameplay elements from the original game, there is a speed boost feature to help the player escape danger. This adds an element of stategy to the game, as the boost can only be used after a cooldown period.

Procedural space diorama

Simulated planetary system with procedurally generated planets and spaceship fleets. Built in Unity and C# (PC).

A 3D diorama demonstrating procedural content generation. Planets, spaceships, UFOs and spiral galaxies are generated using PCG techniques. Parameters such as galaxy shape and ship components were determined randomly, while others such as the placement of fleets and the colour of planets are chosen depending on their position within the overall scene and distance to the star.

The diorama can be viewed from multiple viewpoints using camera controls. This is useful for observing the rotation of planets around the star and the movement of fleets towards UFOs situated at the outer edges of the star system.

Minigolf

A minigolf course game featuring PhysX middleware. Built in C++ (PC).

This game was created for a physics simulation assignment. The golf club can be swung back and forth to apply the desired amount of force to the ball. The club is attached to a PhysX joint and forces are applied when the player provides input.

Rotation of the windmill sails is achieved using a joint and motor. The flag above the hole is created using the PhysX cloth feature and moves dynamically with an applied wind force. There are also several materials that make up the course, identifiable by their colours, and they create different amounts of friction with the ball.
